# 从零到一：我用70分钟打造了一个公众号排版神器

## 引言

今天，我决定动手解决一个困扰我已久的问题：每次写完文章，都要手动调整公众号排版，然后再复制到知乎、小红书等多个平台。这个过程既繁琐又低效。

于是，我花了70分钟，从零开发了一个完整的公众号排版工具——WeChat Formatter。现在，我想把这个过程分享给大家。

---

## 第一步：需求分析

### 痛点是什么？

1. **排版繁琐**：每次都要手动调整字体、颜色、间距
2. **多平台分发**：同一内容要适配5个平台的格式
3. **图片处理**：手动找图、下载、插入，耗时费力
4. **重复劳动**：每次文章都要重复相同的工作

### 解决方案设计

我想做一个工具，能够：
- ✅ 自动解析Markdown文章结构
- ✅ 生成响应式HTML（适配公众号）
- ✅ 自动适配多个平台（知乎、小红书、微博、头条）
- ✅ 自动生成配图（AI）
- ✅ 本地缓存图片，避免重复下载

---

## 第二步：技术选型

### 为什么选这些技术？

| 技术栈 | 原因 |
|--------|------|
| Python | 简单易用，生态丰富 |
| 阿里通义万相API | 图片生成质量高，价格合理 |
| unittest | 测试框架成熟稳定 |
| OpenClaw | 智能代理，易于集成 |

### 为什么不选其他方案？

- **在线工具**：需要登录，有数据泄露风险
- **付费SaaS**：成本高，功能受限
- **现成模板**：不够灵活，难以定制

## 第三步：核心功能开发

### 1. Markdown解析器

```
解析逻辑：
# 标题 → heading元素
段落 → paragraph元素
- 列表 → list元素
```代码``` → code元素
> 引用 → quote元素
![图片](url) → image元素
```

**代码量**：165行

### 2. 公众号排版器

生成响应式HTML，包含：
- 677px最大宽度（公众号推荐）
- 字体大小18px（正文）
- 标题渐变色
- 代码块高亮
- 引用块美化

**代码量**：146行

### 3. 平台适配器

适配5个平台：

| 平台 | 输出格式 | 特点 |
|------|---------|------|
| 公众号 | HTML | 富文本，样式优化 |
| 知乎 | Markdown | 原始格式 |
| 小红书 | Emoji+分段 | 🔸 ✨ 📌 |
| 微博 | 140字摘要 | 自动截断 |
| 头条 | 标题+正文 | 结构化数据 |

**代码量**：101行

### 4. 图片生成器

集成阿里通义万相API：

```python
def generate_image(prompt, style="photography"):
    return {
        "model": "wanx-v1",
        "input": {
            "prompt": prompt,
            "style": style,
            "watermark": False  # 无水印
        }
    }
```

**代码量**：230行

### 5. 图片缓存管理器

基于URL的MD5哈希生成缓存key：

```python
cache_key = md5(url)
local_path = f"cache/images/downloads/{cache_key}.jpg"
```

**代码量**：190行
